# Definition of Done

## A PBI May Be Marked Done Only When

- Its issue-specific acceptance criteria are satisfied.
- The project Definition of Done is satisfied.

## Definition of Done

- All issue acceptance criteria are satisfied.
- The work is reviewed by another team member.
- For user stories, linked supporting PBIs provide the required implementation, review, and verification evidence.
- Required CI checks for the product stack pass.
- Relevant automated tests pass.
- Relevant automated quality requirement tests pass.
- Verification evidence is preserved in normal workflow artifacts.
- `CHANGELOG.md` is updated for every user-visible change, consistent with Product Repository Requirements.
