# Пайплайн распознавания и тегирования ru/tt

Документ описывает алгоритм, прогнанный харнессом `scripts/transcription_quality_test.py` на тестовых
аудио (`tatar only` / `tatar+rus`), результаты и способ масштабирования.

## 1. Архитектура (сквозной путь одного аудио)

Два режима запуска (`python scripts/transcription_quality_test.py tatonly | mixed`) — тяжёлые модели
не уживаются в одном процессе. Татарским файлам LID не нужен (язык форсирован) →
отдельный лёгкий процесс; смешанным нужен LID + Whisper-ru → отдельный процесс.

Татарская ASR — **`yasalma/whisper-finetuned-tt-asr`** (Whisper-small, дообучен на
реальных тат. ASR-датасетах). В отличие от wav2vec2 CTC даёт стандартную орфографию,
пунктуацию и ПРОБЕЛЫ между словами (CTC слипал слова в смешанной речи).

```
mp3 (кириллический путь)
  └─ копия в ASCII-путь         (PyAV не открывает кириллицу)
  └─ decode_audio → 16 кГц моно float32
  └─ VAD (Silero, неагрессивный): режет только паузы > 1.5 c → речевые сегменты
  └─ сегмент дробится на под-окна ≤ 25 c  (память + языковая маршрутизация)
       └─ для каждого под-окна:
            режим tatonly (татарские файлы):
              └─ язык ФОРС tt → Whisper-TT (yasalma/whisper-finetuned-tt-asr)
            режим mixed (смешанные файлы):
              ├─ аудио-LID facebook/mms-lid-256: P(tat) vs P(rus)
              ├─ tt → Whisper-TT
              └─ ru → Whisper large-v3 (language=ru, ленивая загрузка) + антигаллюцинации
       └─ пословный тег языка (§2)
  └─ JSON (слова: text, start, end, conf, seg_lang, lang) + TXT транскрипт
```

Ключевые инженерные моменты (иначе не работает):
- **Два режима**: тяжёлые модели не держим в одном процессе (MMS + CTranslate2
  дают нативные краши/OOM).
- **Порядок загрузки в mixed**: transformers-модели (MMS-LID, Whisper-TT) — первыми,
  Whisper large-v3 (CTranslate2) — лениво/последним. Иначе нативный краш.
  Плюс `KMP_DUPLICATE_LIB_OK=TRUE`, `HF_HOME=D:\hf_cache`.
- **Под-окна ≤ 25 c**: многоминутный сегмент даёт OOM (alloc > 1.3 ГБ).

## 2a. Татарская ASR (Whisper-TT)

`yasalma/whisper-finetuned-tt-asr` транскрибируется через `generate(language="tt")`,
результат разбивается по пробелам на токены (с сохранением пунктуации) для пословного
тегирования. Лёгкий фильтр `TT_HALLUCINATION` отсекает редкие одиночные галлюцинации
на джинглах. История движков: wav2vec2-300M → mms-1b-all → Whisper-TT (каждый шаг —
заметно лучше по орфографии; Whisper-TT добавил пунктуацию и словоразделение).

## 2. Тег языка слова (`backend/src/lang_tag.py`)

Приоритет сигналов (от надёжного к слабому):
1. **Спецбуквы** ә ө ү җ ң һ — есть только в татарском → `tt`.
2. **Лексикон** `backend/src/data/tatar_wordlist.txt` → `tt`
   (ловит татарские слова, записанные ТОЛЬКО русскими буквами: яп, япма, малай…).
3. Иначе — **язык сегмента** от LID/форса.
4. Прочее → `ru` (основной язык корпуса).

## 3. Антигаллюцинации

Whisper на не-русской/тихой речи выдаёт «титры»: «Субтитры создавал…»,
«Продолжение следует», «Спасибо за просмотр». Сегмент с такими маркерами
отбрасывается целиком (список `HALLUCINATION_MARKERS` в `transcription_quality_test.py`).
Главный источник этих галлюцинаций — татарский сегмент, ошибочно ушедший в Whisper-ru;
поэтому форс `tt` на татарских файлах убирает их в принципе.

## 4. Результаты тестов

| Файл | Слов | tt | ru | Тег ru/tt |
|---|---|---|---|---|
| В_метро | 26 | 26 | 0 | ✅ 100% |
| Знакомство | 14 | 14 | 0 | ✅ 100% |
| Кафеда | 21 | 21 | 0 | ✅ 100% |
| Кино | 53 | 53 | 0 | ✅ 100% |
| Хобби | 31 | 31 | 0 | ✅ 100% |
| 30_коротких_фраз | 221 | 169 | 52 | ⚠️ вступление ✅, тело читаемо |
| Практичный (скетч) | 36 | 9 | 27 | ✅ 35/36 (97%) |

- **Тег ru/tt** — основная задача — решён практически идеально, кроме «тела» ролика
  «30 фраз» (быстрое чередование татарской фразы и её русского перевода тем же голосом).
- **Скетч «Практичный»**: 9 форм глагола *ябу* (яп/япма/япманы/япсаны), записанных
  русскими буквами, корректно помечены `tt` — это работает ТОЛЬКО через лексикон.
- **Орфография татарского** (эволюция движков): 300M CTC → mms-1b-all → **Whisper-TT**.
  Whisper-TT добавил пунктуацию и словоразделение: «стонсые»→«Бу нинди станция,»,
  слипшееся «йокларгабармин аңламыйм» → раздельные фразы с «?» и «.».

### Остаточный потолок модели (не баг пайплайна)
Whisper-TT — Whisper-small; остаются мелкие ошибки слов («Мина»←Мин,
«милодрамалар»←мелодрамалар). Тег языка при этом верный. Дальше — более крупный
Whisper-Tatar финетюн или собственное дообучение.

## 5. Как масштабировать на новые татарские слова

1. **Слова со спецбуквами** (ә ө ү җ ң һ) — НИЧЕГО делать не надо: тегируются
   автоматически по орфографии.
2. **Слова русскими буквами** (яп, малай, матур…) — добавить в
   `backend/src/data/tatar_wordlist.txt` (одно слово на строку, строчными).
   Файл автозагружается при импорте `lang_tag`. ⚠️ НЕ добавлять токены, совпадающие
   с русскими словами (бар, кит, без, да) — иначе русские слова станут `tt`.
3. **Качество татарской транскрипции** масштабируется заменой ASR-модели в ветке `tt`
   (сейчас `yasalma/whisper-finetuned-tt-asr` — константа `TT_WHISPER_ID`). Архитектура
   пайплайна не меняется — поменять id на более крупный/лучший Whisper-Tatar.
4. **Надёжность маршрутизации** на смешанной речи — улучшается заменой LID
   (MMS-LID-256 → специализированный ru/tt классификатор) и/или word-level LID.
5. **Новые языки/диктора**: пайплайн расширяется добавлением ветки ASR + языка в
   LID; тег языка — добавлением правил/словника в `lang_tag`.
