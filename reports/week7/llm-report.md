# LLM Usage Report – Week 7

## Overview

During Week 7, the team continued using Large Language Model (LLM) tools to support both frontend and backend development. AI assistance was used to improve the user experience, resolve visual issues, and implement new project functionality. All AI-generated suggestions were reviewed, modified when necessary, and tested before being integrated into the project.

## Tools Used

The team used the following LLM-based tool during Week 7:

- Cursor

## Frontend Development

Cursor was used during frontend development to assist with resolving visual bugs and improving the application's user interface. It also helped implement a new statistics tab that displays speaker-related statistics, providing guidance on UI structure, component organization, and implementation details.

All generated suggestions were reviewed, refined, and tested by the frontend developer before being committed to the repository.

## Backend Development

Cursor was also used during backend development to support the implementation of the speaker statistics functionality. It assisted with backend logic, data processing, and debugging related to generating and serving speaker statistics to the frontend.

The team carefully reviewed and validated all AI-generated suggestions before integrating them into the project.

## Human Responsibility

Cursor was used solely as a development assistant. Team members remained fully responsible for the design, implementation, testing, and validation of all features. Every AI-generated recommendation was reviewed and verified to ensure the correctness, reliability, and quality of the final software.
