# LLM Usage Report – Week 6

## Overview

During Week 6, the team continued using Large Language Model (LLM) tools to support both frontend and backend development. AI assistance was primarily used to speed up feature implementation and solve technical challenges. All AI-generated suggestions were reviewed, modified when necessary, and validated before being integrated into the project.

## Tools Used

The team used the following LLM-based tool during Week 6:

- Cursor

## Frontend Development

Cursor was used during frontend development to assist with implementing project statistics and improving the user interface related to displaying statistical information. It also provided support while patching and integrating the speech transcription functionality into the frontend workflow.

The generated suggestions were treated as recommendations only. All code was reviewed, tested, and refined by the frontend developer before being committed to the repository.

## Backend Development

Cursor was also used during backend development to support the implementation of statistics generation and processing. Additionally, it assisted with patching and improving the speech transcription pipeline, helping identify implementation issues and suggesting possible solutions.

All backend changes were carefully reviewed, tested, and adapted by the team before being merged into the project.

## Human Responsibility

Cursor was used as a development assistant throughout Week 6. Team members remained fully responsible for the design, implementation, testing, and validation of all features. AI-generated suggestions were never accepted without review, ensuring that the final code and project documentation accurately reflected the team's work.
