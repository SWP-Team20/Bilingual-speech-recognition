# LLM Usage Report

## Overview

During Assignment 3, our team used several AI tools to support development and documentation tasks. These tools helped us find solutions faster, understand technical issues, and improve the quality of project documentation. AI tools were used only as assistants. All implementation decisions, coding, testing, and documentation reviews were performed by team members.

## Frontend Development

**Tools Used:**

* Google Gemini
* Claude

### How They Were Used

The frontend developer used Gemini mainly when selecting technologies and solving implementation problems. It was helpful for comparing different approaches and understanding framework-related issues.

Claude was used throughout frontend development to discuss implementation ideas, investigate bugs, and suggest possible solutions when development problems occurred.

In both cases, the generated suggestions were treated as recommendations rather than final answers. Code was reviewed, modified when necessary, and tested before being added to the project.

## Backend Development

**Tool Used:**

* Google Gemini
* Claude

### How It Was Used

Claude and Gemini was used during backend development mostly for debugging purposes. When errors occurred, it helped explain error messages, identify possible causes, and suggest troubleshooting steps.

The backend implementation itself was developed and tested by the team member responsible for that part of the project. AI suggestions were used only when additional guidance was needed during problem solving.

## Documentation and Reporting

**Tool Used:**

* ChatGPT

### How It Was Used

ChatGPT was used while preparing project documentation and assignment reports. It helped review documents for clarity, grammar, and consistency. It was also used to check whether required sections were missing and to improve the structure of written reports.

All generated text was reviewed and edited before being included in the repository. The team verified that the final documentation accurately reflected the work completed during the Sprint.

## Reflection on AI Usage

The team found AI tools useful for reducing time spent on debugging and improving documentation quality. However, AI-generated suggestions were not accepted automatically. Team members reviewed recommendations, tested proposed solutions, and made the final decisions themselves.

AI tools supported the development process, but responsibility for the final product, source code, testing results, and project documentation remained entirely with the team.
